# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please email security@example.com with details about the vulnerability. Do not open a public GitHub issue for security vulnerabilities.

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Supported Versions

We release patches for security vulnerabilities as follows:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| 0.x     | :x:                |

## Security Updates

Security updates will be released as soon as possible after a vulnerability is confirmed. We will provide a 90-day advance notice for end-of-life security support.

## Best Practices

- Keep your dependencies up to date
- Use environment variables for sensitive data
- Report security issues responsibly
- Enable two-factor authentication on your GitHub account

## Contact

For security concerns, please contact the maintainers privately rather than using public issues.
